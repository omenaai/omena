## Introduction

OMENA is an Agent Intelligence Layer that transforms raw on-chain blockchain activity into structured intelligence. The current public release focuses on providing token intelligence through a dashboard experience designed to make blockchain information easier to interpret.

This documentation covers the current public direction of OMENA. It is intended as a foundation that can be expanded as the product evolves.

<span id="vision" />

## Vision

OMENA aims to become foundational intelligence infrastructure for AI-powered blockchain applications by organizing raw data into contextual insights.

<span id="why-omena" />

## Why OMENA Exists

On-chain data is publicly available but not inherently readable. Blockchain explorers surface raw transactions. Charting tools surface price. Neither surface intent.

OMENA exists to close that gap — to take the noise of on-chain activity and return a signal. One that is structured enough for machines to act on, and clear enough for people to understand.

<span id="dashboard-beta" />

## Dashboard Beta

The Dashboard Beta accepts a supported token contract address, gathers public blockchain information, processes intelligence modules, and returns a structured report.

### How It Works

1. User opens `/app`
2. User pastes a supported token contract address
3. OMENA validates the address format
4. OMENA gathers public blockchain information
5. OMENA processes intelligence modules
6. OMENA returns a structured report

The output is informational. It is not financial advice.

<span id="architecture" />

## Architecture

```
Data Collection
  → Normalization
  → Risk Intelligence
  → Behavior Intelligence
  → Signal Intelligence
  → Context Assembly
  → AI Summary
  → Structured Report
```

### Layer 1 — Web App

- Next.js App Router frontend
- Landing page
- Dashboard routes (`/app`, `/app/analyze`)
- Documentation pages

### Layer 2 — Data and Analysis

- DexScreener market snapshot
- Optional Helius enrichment
- Modular intelligence functions
- Mock fallback when live data is incomplete

### AI Summary

Transforms structured intelligence into concise explanations for easier understanding. It is informational and not financial advice.

<span id="risk-intelligence" />

## Risk Intelligence

Evaluates observable indicators such as liquidity characteristics, holder distribution, deployer activity, and metadata consistency.

### Core Capabilities

- Liquidity quality scoring
- Pair age and market depth checks
- Volume versus liquidity stress checks
- Holder concentration review
- Deployer activity indicators
- Plain-language risk summary

### Output

```json
{
  "level": "Low | Medium | High",
  "score": 0,
  "summary": "Readable explanation",
  "flags": [],
  "liquidityHealth": "string",
  "holderConcentration": "string"
}
```

<span id="behavior-intelligence" />

## Behavior Intelligence

Analyzes publicly visible wallet and token behavior to provide additional context.

### Core Capabilities

- Buy versus sell activity interpretation
- Accumulation versus distribution status
- Deployer visibility note
- Graceful partial-data handling

<span id="signal-intelligence" />

## Signal Intelligence

Highlights notable on-chain events and combines them with other intelligence layers.

### Core Capabilities

- 24h price movement review
- Volume abnormality checks
- Transaction flow shifts
- Severity classification: Normal, Watch, Alert

<span id="context-intelligence" />

## Context Intelligence

Assembles module outputs into a final structured report.

### Core Capabilities

- Merge module outputs into one report
- Keep language readable and non-hype
- Produce final verdict and caution level
- Make report easy to copy or share

<span id="faq" />

## FAQ

### What makes OMENA different?

OMENA focuses on intelligence output, not trading execution. It organizes raw on-chain data into contextual, structured insights.

### Does OMENA execute transactions?

No. OMENA is an intelligence layer only.

### Does OMENA require API keys?

No. The dashboard works with mock fallback when live coverage is incomplete.

### Which live data source is active first?

DexScreener is the first live source.

### Is OMENA financial advice?

No. All output is informational only and should not be interpreted as financial advice.

### Which chains are supported?

Currently Solana. Multi-chain support is planned.
